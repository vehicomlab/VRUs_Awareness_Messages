Set-up:
- SPAN: Novatel PwrPak7 was used (https://novatel.com/products/gnss-inertial-navigation-systems/combined-systems/pwrpak7-e2) to collect GNSS measurement (geodetic antenna mounted on a mast) and INS data.
The data was post-processed using the Novatel Inertial Explorer Software (https://novatel.com/products/waypoint-post-processing-software/inertial-explorer). In particular:
	- the GNSS measurements were processed in Real Time Kinematic (RTK) mode using as a base station a static receiver with an antenna located on the roof of the lab.
	- the INS measurements were integrated with the GNSS using a tight-coupling approach.

Lever-arms and orientation:
IMU to GNSS Antenna Lever Arms:
             x=-0.115, y=0.420, z=1.200 m (x-right, y-fwd, z-up)
Body to IMU Rotations:
             xRot=0.000, yRot=0.000, zRot=0.000 degrees
IMU->Secondary Sensor Lever Arms: 
            x=0.170, y=0.000, z=0.070 m (x-right, y-fwd, z-up, IMU->SENSOR)


Folder structure:

data_JRC
└───ref_trajectory
   ├───IMU_raw_data  -> contains raw IMU measurements (velocity, acceleration, heading)
   ├───Mikrotik_solution  -> contains the reference trajectory centred on the Mikrotik antenna.
   └───SPAN_solution      -> contains the reference trajectory centred on the SPAN

Reference trajectory: *_imu.txt contains the velocity and acceleration info + heading
		      *.gga contains the NMEA GGA messages
		      *.txt contains more detailed info on the reference trajectory (lat, lon, head, vel and standard deviation)
